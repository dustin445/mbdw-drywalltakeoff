Google Drive Integration

NEW   googleApi.js

Self-contained module. Exports:

  • getAccessToken() — OAuth 2.0 via Google Identity Services; token is cached
  • createDriveFolder(name) — creates a sub-folder in PARENT_FOLDER_ID; duplicate-safe
  • logToSpreadsheet({...}) — appends a row: Timestamp | Job Name | Job # | Job ID | Folder Name | Folder ID | Folder URL
  • uploadFileToDrive(folderId, filename, content) — multipart JSON upload
  • provisionJob(job) — orchestrates folder creation + sheet logging

Credentials live in GOOGLE_CONFIG at the top of the file. No rebuild needed to update them.

CHANGED   app.jsx

Import

import { provisionJob, uploadFileToDrive } from "./googleApi.js";

initJob — driveFolder field

Every new job object now includes driveFolder: null, populated with { folderId, folderUrl, folderName } after provisioning.

New state: driveFailedJob, driveRetrying

driveFailedJob holds the job if provisioning failed and drives the retry modal. driveRetrying disables the Retry button while in flight.

createJob — async + Drive provisioning

Made async. After navigating (so UI is never blocked), calls provisionJob in the background. Success → toast. Failure → retry modal.

retryDriveProvisioning()

Re-attempts provisioning for the failed job. Keeps modal open until it succeeds.

handleSaveToGoogle()

Serializes the current job to JSON and uploads it to its Drive folder. Filename: Job_Name_YYYY-MM-DD_HH-MM.json. Every save is a new timestamped snapshot.

☁ SAVE TO GOOGLE button

Added below ⬆ IMPORT NEW PRICING in the job screen header. Full-width, indigo.

Drive failure retry modal

Full-screen, red-bordered. Cannot be dismissed — user must retry or fix connectivity.
