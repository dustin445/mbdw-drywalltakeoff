Google Drive Integration

How It Works

1 — Create a new job

After tapping Create Job, the app navigates immediately. In the background:

  • A Google OAuth popup asks you to sign in (first time only)
  • A sub-folder is created in the parent Drive folder, named Job Name __ jobId
  • A row is appended to the Google Sheet: Timestamp | Job Name | Job # | Job ID | Folder Name | Folder ID | Folder URL
  • Toast confirms: "Google Drive folder created"
2 — Save to Google

The ☁ SAVE TO GOOGLE button is in the job screen header, below ⬆ IMPORT NEW PRICING.

  • Serializes the full job to JSON
  • Uploads it to the job's linked Drive folder
  • Filename: Job_Name_YYYY-MM-DD_HH-MM.json — every save is a new timestamped snapshot
3 — Retry modal

If Drive provisioning fails when creating a job, a red-bordered modal appears. It cannot be dismissed — tap Retry until it succeeds, or fix the underlying issue first.

Troubleshooting

OAuth popup never appears Google Identity Services script not loaded — check the browser console
Auth fails Client ID mismatch, or the app's domain is not listed in Authorized JavaScript Origins in Google Cloud Console
Folder not created PARENT_FOLDER_ID is wrong, or the signed-in account doesn't have access to that folder
Sheet not updated SPREADSHEET_ID or SHEET_NAME is wrong
Save to Google fails Job has no linked Drive folder — provisioning never succeeded for this job; create a new job to test

Configuration

All credentials are in googleApi.js under GOOGLE_CONFIG. No rebuild needed after editing.

  • CLIENT_ID — OAuth 2.0 Client ID from Google Cloud Console
  • PARENT_FOLDER_ID — Drive folder where job sub-folders are created
  • SPREADSHEET_ID — Google Sheet where jobs are logged
  • SHEET_NAME — tab name inside that spreadsheet (default: TAKEOFF_DATA)
EXIT Button
Where

Home screen header, top right — beside the ⚙️ Admin Pricing button.

What it does

Returns to the password screen for the current session. Does not clear localStorage — the next page load will auto-login as before.

To permanently clear the session so the password is required on next load, open the browser console and run:

localStorage.removeItem("takeoff_unlocked")
